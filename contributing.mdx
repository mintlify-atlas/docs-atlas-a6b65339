MidOS is community-first. If you have production-tested patterns, battle scars, or discovered that a popular claim is false — we want it.

## How to Contribute Knowledge

Every piece of knowledge in MidOS goes through a rigorous 5-layer validation pipeline. Your contributions help build better AI agents for everyone.

### Step 1: Search First

Before submitting new knowledge, check if it already exists:

```bash
search_knowledge("your topic")
```

This helps us avoid duplicates and lets you build on existing work.

### Step 2: Propose Your Pattern

[Open an issue](https://github.com/MidOSresearch/midos/issues/new) describing:

- **What**: The pattern, discovery, or myth you want to contribute
- **Why**: What makes it battle-tested or empirically verified
- **Evidence**: Code examples, benchmarks, or production experience
- **Stack**: Which technology/framework it applies to

### Step 3: Review Process

Our team will:

1. **Validate** the claim with empirical evidence
2. **Cross-reference** against existing knowledge
3. **Test** the pattern in our validation pipeline
4. **Categorize** it in the appropriate layer:
   - **Chunks** (L1): Curated, indexed knowledge
   - **Skills** (L2): Stack-specific skill packs
   - **Truth** (L3): Empirically verified patches
   - **EUREKA** (L4): Validated improvements with ROI
   - **SOTA** (L5): Best-in-class, currently unimprovable

## Community-First Philosophy

MidOS is built **by developers, for agents**. We believe:

- **Quality over quantity**: Every chunk is reviewed and validated
- **Evidence over opinions**: Empirical data beats anecdotes
- **Battle-tested over trendy**: Production experience matters
- **Myth-busting over repeating**: We verify claims, not copy-paste them

## Knowledge Pipeline

Your contribution flows through our 5-layer pipeline:

```
staging/ → chunks/ → skills/ → truth/ → EUREKA/ → SOTA/
 (entry)    (L1)      (L2)      (L3)     (L4)      (L5)
```

- **46,000+ chunks**: Curated, indexed knowledge
- **200+ skills**: Organized, actionable, versioned by stack
- **50 truth items**: Verified with empirical evidence
- **383 EUREKA discoveries**: Validated improvements with measured ROI
- **140 SOTA items**: Best-in-class, currently unimprovable

## Contributing Code

Interested in contributing to the MidOS codebase?

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

We review PRs regularly and appreciate:
- Clear commit messages
- Tests for new features
- Documentation updates
- Following our code style (Ruff for Python)
