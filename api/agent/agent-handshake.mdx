# agent_handshake

Call this FIRST when connecting to MidOS. Declare your environment and get optimal configuration tailored to your model, client, stack, and project goals.

## Overview

`agent_handshake` is the entry point for all agents connecting to MidOS. It analyzes your capabilities and environment, then returns:

- **Model detection**: Auto-detects context window from 31+ models in catalog
- **Context optimization**: Adjusts response size based on your budget (small/medium/large)
- **Stack-aware recommendations**: Suggests relevant skills and tools for your languages/frameworks  
- **Client-specific tips**: Tailored advice for Claude Code, Cursor, Windsurf, Cline, etc.
- **Adaptive guardrails**: Rules that match your model's capabilities

## Function Signature

```python
async def agent_handshake(
    model: str = "",
    context_window: int = 0,
    client: str = "",
    languages: str = "",
    frameworks: str = "",
    platform: str = "",
    project_goal: str = "",
) -> str
```

## Parameters

### `model` (optional)

**Type:** `str`  
**Default:** `""`

Your model ID. Supports:
- Full names: `"claude-opus-4-6"`, `"gpt-5.3-codex"`, `"gemini-2.5-pro"`
- Short names: `"opus"`, `"sonnet"`, `"haiku"`  
- Aliases: `"o3"`, `"deepseek"`, `"qwen coder"`
- OpenRouter prefixes: `"openrouter/glm-4.5-air:free"` → resolves to `glm-4.5-air`

**Auto-detection logic:**
1. Exact match in MODEL_CATALOG (31 models)
2. Alias lookup (60+ variations)
3. Substring matching (handles prefixed names)
4. Fuzzy matching (0.85 cutoff to prevent wrong matches)

If no match, returns safe defaults (128K context assumed).

**Example models:**
- Claude: `opus`, `sonnet`, `haiku`
- GPT: `gpt-5.3-codex`, `gpt-5.2-xhigh`, `o3`, `o3-mini`
- Gemini: `gemini-2.5-pro` (1M context), `gemini-2.5-flash` (506 t/s)
- DeepSeek: `deepseek-r1`, `deepseek-v3.1`
- Free tier: `glm-4.5-air`, `qwen3-coder`, `llama-3.3-70b`, `big-pickle`

### `context_window` (optional)

**Type:** `int`  
**Default:** `0` (auto-detect)

Your context window in tokens. Pass `0` to auto-detect from model catalog.

**When to set manually:**
- Your model isn't in the catalog
- You want to override the default (e.g., testing with reduced context)
- Your client enforces a lower limit than the model

**Context budget tiers:**
- **small** (≤32K): Compact responses, top 2 tools/skills
- **medium** (32K-128K): Standard format, top 5 tools/skills  
- **large** (>128K): Full format with previews, top 15 skills

### `client` (optional)

**Type:** `str`  
**Default:** `""`

Your CLI/IDE. Recognized clients:
- `"claude-code"`, `"codex-cli"`, `"cursor"`, `"windsurf"`
- `"cline"`, `"continue"`, `"aider"`, `"zed"`  
- `"github-copilot"`, `"amazon-q"`, `"replit"`, `"lovable"`
- `"opencode"`

**Client detection:**
- Resolves aliases: `"claude code"` → `"claude-code"`, `"gh-copilot"` → `"github-copilot"`
- Loads CLI-specific profile from `config/cli_profiles.json`
- Applies tool restrictions, delegation policies, attention pinch rules

**Why it matters:**
- **Hooks**: Claude Code, Windsurf → lifecycle hooks enabled
- **Memory**: Windsurf, GitHub Copilot → session persistence tips
- **Background agents**: Claude Code, Zed → parallel delegation advice
- **Context management**: Cursor (dynamic pruning), Aider (repo-map)

### `languages` (optional)

**Type:** `str` (comma-separated)  
**Default:** `""`

Programming languages for your project.

**Examples:**
- `"python,typescript"`
- `"rust,go"`  
- `"javascript,html,css"`

**Impact:**
- Filters skills by `compatibility.json` (exact match gets +3 score)
- Suggests stack-specific skills (e.g., `python` → `django_v5`, `fastapi_patterns`)
- Ranks tools by language keywords (e.g., `typescript` boosts `chunk_code`)

### `frameworks` (optional)

**Type:** `str` (comma-separated)  
**Default:** `""`

Frameworks/libraries you're using.

**Examples:**
- `"fastapi,react"`
- `"django,postgresql,redis"`
- `"nextjs,tailwind,prisma"`

**Impact:**
- Matches against skill `compatibility.json` (exact match +3 score)
- Suggests framework-specific skills:
  - `react` → `react_comprehensive`  
  - `nextjs` → `nextjs`
  - `fastapi` → `fastapi_patterns`
  - `postgresql` → `postgresql_patterns`

### `platform` (optional)

**Type:** `str`  
**Default:** `""`

Your operating system.

**Examples:**
- `"windows"`, `"linux"`, `"macos"`

**Impact:**
- Adds platform-specific guardrails:
  - **Windows**: `Use ';' not '&&'`, `Use 'python -c' as fallback for PowerShell`
  - **Linux/macOS**: Standard bash workflows

### `project_goal` (optional)

**Type:** `str`  
**Default:** `""`

What you're working on (free-form description).

**Examples:**
- `"manga engine with SVG rendering"`
- `"real-time chat app with WebSocket"`  
- `"microservices API with gRPC"`

**Impact:**
1. **Semantic search**: Finds relevant knowledge chunks via vector similarity (top 5, score ≥0.25)
2. **Skill matching**: Scores skills by keyword overlap with goal (e.g., "API" → `fastapi_patterns`)
3. **Tool ranking**: Boosts tools that match goal keywords

**Note:** Generic goals (`"test"`, `"hello"`, `"demo"`) are skipped to avoid noise.

## Context Window Optimization

**Auto-detection priority:**

1. **profile.context_window** (explicit override)
2. **model_spec.context_window** (from MODEL_CATALOG)  
3. **client_spec.max_context** (from CLIENT_CATALOG)
4. **Default:** 128,000 tokens

**Effective window** = `min(all_detected_windows, 10_000_000)`

**Response tier logic:**

```python
if effective_window <= 32_000:
    tier = "small"  # Compact: top 2 tools, 2 skills, 3 guardrails
elif effective_window <= 128_000:
    tier = "medium"  # Standard: top 5 tools, 5 skills, all guardrails
else:
    tier = "large"  # Full: top 15 skills, 5 chunks with previews, all tips
```

**Client overrides:**

- **Windsurf:** max_context = 300,000 (overrides model if lower)
- **Claude Code:** max_context = 200,000  
- **Cline:** max_context = 0 (model-dependent, auto-truncation)
- **Cursor:** max_context = 200,000 (dynamic pruning)

## Troubleshooting

**Q: Model not detected?**  
A: Pass `context_window=N` manually. Your model will work but get generic recommendations.

**Q: Skills seem irrelevant?**  
A: Add `languages` and `frameworks` parameters for stack-aware filtering.

**Q: Response too large for my context?**  
A: Pass accurate `context_window` — handshake will compress output for small contexts.

**Q: Want tool X but it's not in my tier?**  
A: Check `min_tier` in response. Community tier gets 17 tools, dev tier gets all tools.

**Q: Client not recognized?**  
A: You'll get generic defaults (read-only safe). Pass client name for tailored config.
