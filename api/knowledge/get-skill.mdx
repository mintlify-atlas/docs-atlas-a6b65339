## Overview

The `get_skill` tool retrieves a specific skill/capability document by name. Skills contain specialized knowledge, patterns, and best practices for various technologies, frameworks, and development workflows.

## Function Signature

```python
async def get_skill(name: str) -> str
```

## Parameters

<ParamField path="name" type="string" required>
  The skill name to retrieve. Case-insensitive matching is supported.
  
  Examples:
  - `"react"` - React framework patterns
  - `"fastapi"` - FastAPI best practices
  - `"RAG_SYSTEMS_2026_SOTA"` - State-of-the-art RAG patterns
  - `"docker-containerization"` - Docker containerization guide
</ParamField>

## Return Value

Returns a string containing:
- **Success**: Full content of the skill document in Markdown format
- **Not Found**: Error message with list of available skills (up to 20)

```
Skill not found: {name}

Available skills: ['react', 'fastapi', 'docker-containerization', ...]
```

## Behavior

1. **Exact Match**: First attempts to find an exact match for `{name}.md`
2. **Case-Insensitive Search**: If no exact match, performs case-insensitive search
3. **Error with Suggestions**: Returns helpful error with available skill names

## Usage Examples

### Retrieve a React Skill

```python
result = await get_skill(name="react")
# Returns full React patterns and best practices document
```

### Get FastAPI Patterns

```python
result = await get_skill(name="fastapi")
# Returns FastAPI skill with API design patterns
```

### Case-Insensitive Lookup

```python
# These are equivalent:
result = await get_skill(name="Docker-Containerization")
result = await get_skill(name="docker-containerization")
result = await get_skill(name="DOCKER_CONTAINERIZATION")
```

### Handle Missing Skills

```python
result = await get_skill(name="nonexistent")
# Returns: "Skill not found: nonexistent\n\nAvailable skills: [...]"
```
