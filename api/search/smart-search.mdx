## Overview

`smart_search` automatically selects the optimal search strategy (keyword, semantic, or hybrid) based on query analysis. It removes the cognitive overhead of choosing between search tools.

**When to use**: When you want the system to automatically pick the best search approach, or when building agent workflows that need to handle diverse query types.

<Note>
`smart_search` is a **conceptual pattern** demonstrated in MidOS but not exposed as a standalone MCP tool. This page documents how to implement smart search logic in your own integrations.
</Note>

## Decision Logic

The smart search algorithm analyzes query characteristics to pick the best mode:

```python
def smart_search(query: str, top_k: int = 5) -> List[Dict]:
    """Auto-detect optimal search mode based on query."""
    
    # 1. Code/identifier queries → keyword search
    if is_code_query(query):
        return search_memory(query, top_k, search_mode="keyword")
    
    # 2. Natural language questions → semantic search
    if is_conceptual_query(query):
        return search_memory(query, top_k, search_mode="vector")
    
    # 3. Mixed/ambiguous queries → hybrid search
    return search_memory(query, top_k, search_mode="hybrid", rerank=True)
```

## Detection Rules

### Code Queries → Keyword Mode

**Triggers**:
- Contains code symbols: `(`, `)`, `{`, `}`, `[`, `]`, `;`
- Starts with code keywords: `import`, `def`, `class`, `function`, `const`, `let`
- Contains camelCase or snake_case identifiers
- File extensions: `.py`, `.js`, `.tsx`, `.md`

**Examples**:
```python
# These trigger keyword search
"useEffect cleanup function"
"FastAPI.Depends(get_db)"
"import React from 'react'"
"RAG_SYSTEMS_2026_SOTA.md"
"error_boundary.tsx"
```

**Rationale**: Code identifiers need exact matching. Semantic embeddings may not preserve special characters or casing.

### Conceptual Queries → Semantic Mode

**Triggers**:
- Starts with question words: `how`, `what`, `why`, `when`, `where`, `which`
- Contains qualifiers: `best practice`, `pattern for`, `approach to`, `strategy for`
- Natural language sentence structure (>10 words)
- No code symbols

**Examples**:
```python
# These trigger semantic search
"how to implement rate limiting in web APIs"
"what are the best practices for React state management"
"explain the difference between client and server components"
"strategies for handling authentication in microservices"
```

**Rationale**: These queries benefit from conceptual understanding and synonym matching.

### Mixed Queries → Hybrid Mode

**Triggers**:
- Contains both technical terms AND natural language
- Ambiguous intent
- Short queries (less than 5 words) without clear code patterns
- Doesn't match code or conceptual rules

**Examples**:
```python
# These trigger hybrid search
"FastAPI rate limiting"
"React error boundaries best practices"
"implement caching"
"authentication patterns"
```

**Rationale**: These need both keyword precision (technical terms) and semantic understanding (concepts).

```python Detection Functions
import re

def is_code_query(query: str) -> bool:
    """Detect if query is code/identifier search."""
    code_symbols = ["(", ")", "{", "}", "[", "]", ".", ";"]
    code_keywords = ["import", "def", "class", "function", "const", "let", "var"]
    
    # Check for code symbols
    if any(sym in query for sym in code_symbols):
        return True
    
    # Check for code keywords at start
    if any(query.lower().startswith(kw) for kw in code_keywords):
        return True
    
    # Check for camelCase/snake_case
    if re.search(r'[a-z][A-Z]|[a-z]_[a-z]', query):
        return True
    
    # Check for file extensions
    if re.search(r'\.(py|js|tsx?|jsx|md|json|ya?ml)\b', query):
        return True
    
    return False

def is_conceptual_query(query: str) -> bool:
    """Detect if query is conceptual/natural language."""
    question_words = ["how", "what", "why", "when", "where", "which"]
    qualifiers = ["best practice", "pattern for", "approach to", "strategy for"]
    
    q_lower = query.lower()
    
    # Starts with question word
    if any(q_lower.startswith(qw) for qw in question_words):
        return True
    
    # Contains qualifiers
    if any(qual in q_lower for qual in qualifiers):
        return True
    
    # Long natural language sentence
    if len(query.split()) > 10 and not is_code_query(query):
        return True
    
    return False
```

```python Smart Search Function
from hive_commons.vector_store import search_memory

def smart_search(query: str, top_k: int = 5, stack: str = "") -> List[Dict]:
    """Automatically select optimal search mode."""
    
    # Determine mode
    if is_code_query(query):
        mode = "keyword"
        rerank = False
    elif is_conceptual_query(query):
        mode = "vector"
        rerank = False
    else:
        mode = "hybrid"
        rerank = True  # Hybrid benefits from reranking
    
    # Execute search
    results = search_memory(
        query=query,
        top_k=top_k,
        search_mode=mode,
        rerank=rerank
    )
    
    # Optional: Apply stack filtering post-search
    if stack and results:
        results = apply_stack_boost(results, stack)
    
    return results

def apply_stack_boost(results: List[Dict], stack: str) -> List[Dict]:
    """Boost results mentioning stack technologies."""
    tokens = [t.strip().lower() for t in stack.split(",")]
    
    for r in results:
        text_lower = r["text"].lower() + " " + r["source"].lower()
        boost = sum(1 for t in tokens if t in text_lower)
        r["score"] = r.get("score", 0) + (boost * 0.1)
    
    results.sort(key=lambda x: x["score"], reverse=True)
    return results
```

### Integration Example

```python
# Agent workflow with smart search
class AgentWorkflow:
    def answer_question(self, user_query: str, context: Dict) -> str:
        # Extract stack from context
        stack = ",".join(context.get("languages", []) + context.get("frameworks", []))
        
        # Smart search automatically picks best mode
        results = smart_search(
            query=user_query,
            top_k=5,
            stack=stack
        )
        
        # Format results for LLM
        knowledge = "\n\n".join([
            f"Source: {r['source']}\n{r['text'][:500]}"
            for r in results
        ])
        
        return self.llm.complete(f"Question: {user_query}\n\nKnowledge:\n{knowledge}")
```

## Performance Impact

### Detection Overhead

- **Regex checks**: less than 1ms
- **String operations**: less than 1ms
- **Total overhead**: less than 5ms (negligible)

### Search Mode Latency

| Detected Mode | Avg Latency | Use Case |
|---------------|-------------|----------|
| Keyword | 10-30ms | Code identifiers |
| Semantic | 50-1200ms | Conceptual queries |
| Hybrid + Rerank | 100-600ms | Mixed queries |

**Smart search adds minimal overhead while optimizing search quality.**

## Real-World Workflow

```python
class MidOSAgent:
    def __init__(self, stack: List[str]):
        self.stack = ",".join(stack)
    
    def research(self, topic: str) -> str:
        """Research topic using smart search."""
        # Smart search picks best mode automatically
        results = smart_search(
            query=topic,
            top_k=10,
            stack=self.stack
        )
        
        if not results:
            # Fallback: try broader query
            fallback_query = topic.split()[0]  # First word only
            results = smart_search(fallback_query, top_k=5)
        
        return self.format_results(results)
    
    def lookup_code(self, identifier: str) -> str:
        """Quick code lookup (forced keyword mode)."""
        # Override smart detection for guaranteed keyword search
        results = search_memory(
            query=identifier,
            top_k=3,
            search_mode="keyword"
        )
        return self.format_results(results)
```

## Limitations

<Warning>
**Smart search is heuristic-based**, not ML-powered. Detection rules can be fooled by:

- Ambiguous queries: "React" (framework or verb?)
- Mixed intent: "import best practices" (code keyword + conceptual)
- Domain-specific jargon: May not recognize niche technical terms

**Mitigation**: Provide explicit `search_mode` override when needed.
</Warning>
