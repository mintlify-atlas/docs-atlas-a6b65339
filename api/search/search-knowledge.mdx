## Overview

Performs keyword-based search across all knowledge layers in MidOS. This is the fastest search method, ideal for finding exact matches or known terms.

**When to use**: You know specific keywords or file names you're looking for. For semantic understanding, use `semantic_search` instead.

## Function Signature

```python
async def search_knowledge(query: str, max_results: int = 5) -> str
```

## Parameters

<ParamField path="query" type="string" required>
  Search query with keywords or topics. Searches both file names and content.
  
  **Examples**: "RAG implementation", "caching patterns", "React hooks"
</ParamField>

<ParamField path="max_results" type="integer" default="5">
  Maximum number of results to return. Higher values may increase response time.
  
  **Range**: 1-50 recommended
</ParamField>

## Return Format

Returns a formatted string with search results:

```markdown
Found {count} results for '{query}':

## React 19 Server Components
Path: knowledge/chunks/react19_server_components.md
```
Server Components in React 19 allow you to render components on the server...
```
```

If no results found:
```
No results found for: your query
```


## Search Behavior

### What Gets Searched

- **File names**: Case-insensitive match on file stems
- **File content**: Full-text search across all markdown files
- **Knowledge layers**: Skills, protocols, EUREKA, truth patches, chunks

### Result Ordering

1. Files with query in name (ranked first)
2. Files with query in content (by discovery order)
3. Limited to `max_results` total

### Context Snippets

- **Match found**: 100 chars before + 200 chars after the match
- **Name match only**: First 300 chars of file content
- Snippet length: ~300 characters

## Performance

- **Latency**: less than 100ms typical (no API calls)
- **No rate limits**: Pure file system search
- **Scales**: Handles 46K+ chunks efficiently

## Stack Filtering

`search_knowledge` does **not** support stack filtering. For stack-aware search, use:

- `semantic_search(query, stack="python,react")`
- `list_skills(stack="python,react")`

## Error Handling

```python
# Empty query
result = await search_knowledge("")
# Returns: "No results found for: "

# No matches
result = await search_knowledge("xyzabc123nonexistent")
# Returns: "No results found for: xyzabc123nonexistent"

# File system errors are handled gracefully (OSError caught)
```
