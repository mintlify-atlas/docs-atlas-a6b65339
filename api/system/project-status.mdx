## Function Signature

```python
async def project_status() -> str
```

## Description

Returns a comprehensive, live MidOS system dashboard with real-time knowledge base statistics, available MCP tools with usage examples, research queue status, and agent quick-start guide.

This is your `/status` command - use it to orient yourself and teach agents how to leverage MidOS effectively.

The dashboard includes:
- **Real-time knowledge stats**: Chunks (L1), Skills (L2), Truth Patches (L3), EUREKA (L4)
- **Vector store health**: Status and total vector count
- **Recent knowledge**: Last 5 chunks added by modification time
- **Research queue**: Pending topics from RESEARCH_INTEREST_QUEUE.md
- **MCP tools guide**: All available tools with arguments and descriptions
- **Quick-start boot sequence**: Recommended workflow for new agents
- **Pro tips**: Best practices for maximizing MidOS effectiveness

## When to Use

Use `project_status` when you need to:
- **Orient a new agent** - Call this first to get the full system overview
- **Check knowledge base health** - See live counts across all layers
- **Discover available tools** - Get the complete MCP tools catalog with examples
- **Learn the workflow** - Understand the recommended agent boot sequence
- **Monitor system growth** - Track vectors, chunks, and skills over time
- **Debug research pipeline** - See what's queued for processing

This should be the **first tool you call** in any MidOS session to understand what's available and how to use it.

## Parameters

This tool takes no parameters.

## Response Structure

Returns a formatted Markdown string with the following sections:

### 1. Knowledge Base (live)

<ResponseField name="Chunks (L1)" type="integer">
  Number of markdown files in `knowledge/chunks/`
</ResponseField>

<ResponseField name="Skills (L2)" type="integer">
  Number of skill directories with SKILL.md files
</ResponseField>

<ResponseField name="Truth Patches (L3)" type="integer">
  Number of markdown files in `knowledge/truth/`
</ResponseField>

<ResponseField name="EUREKA (L4)" type="integer">
  Number of markdown files in `knowledge/EUREKA/`
</ResponseField>

<ResponseField name="Staging queue" type="integer">
  Number of files waiting in `knowledge/staging/`
</ResponseField>

<ResponseField name="Vector store" type="string">
  Status (online/offline/error) and total vector count
</ResponseField>

### 2. Recent Knowledge

List of the 5 most recently modified chunk files by modification time.

### 3. Research Queue

Pending topics extracted from the `RESEARCH_INTEREST_QUEUE.md` PENDING section.

### 4. MCP Tools

Complete catalog of available tools with:
- Tool name
- Arguments
- Description and use case

### 5. Quick-Start Boot Sequence

Recommended 4-step workflow for agents:
1. `agent_handshake` - Get personalized config
2. `semantic_search` - Check existing knowledge
3. `list_skills` - Find reusable skills for your stack
4. `episodic_store` - Share learnings back to MidOS

### 6. Pro Tips

Best practices including:
- Always search before building
- Use stack filters for relevance
- Store your learnings for the knowledge loop
- EUREKA atoms contain battle-tested improvements
- Handshake once per session for optimization

## Example Usage

```python
# Get full system dashboard
dashboard = await project_status()
print(dashboard)

# Use at start of agent session
async def agent_init():
    # First, get the system status
    status = await project_status()
    
    # Parse to understand available tools
    print("System Overview:")
    print(status)
    
    # Then proceed with handshake
    config = await agent_handshake(
        model="claude-sonnet-4.5",
        client="cline",
        languages=["python", "typescript"],
        project_goal="Build FastAPI backend"
    )
```
