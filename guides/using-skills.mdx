MidOS organizes curated knowledge into **skill packs**: structured, validated patterns for specific technologies and frameworks. Each skill is battle-tested in production projects.

## What Are Skills?

Skills are comprehensive guides that include:
- **Production patterns**: Real-world implementations, not just theory
- **Code examples**: Copy-paste-ready snippets
- **Best practices**: Validated by empirical evidence
- **Common pitfalls**: Mistakes to avoid
- **Version-specific**: Tagged with framework versions (e.g., React 19, Django 5)

### Skill Structure

```
skills/
├── react_19_comprehensive/
│   ├── SKILL.md                  # Main guide
│   ├── compatibility.json        # Supported stacks
│   ├── examples/                 # Code samples
│   └── tests/                    # Validation tests
└── fastapi_patterns/
    ├── SKILL.md
    └── compatibility.json
```

Each skill includes:
- **SKILL.md**: Complete documentation
- **compatibility.json**: Languages, frameworks, versions
- **examples/**: Working code samples
- **tests/**: Validation and test patterns

## Browsing Skills

### List All Skills

Get a complete inventory:

```python
list_skills()
```

**Output:**
```

- django_v5_best_practices

- kubernetes_orchestration
- typescript_mastery
...
```

### Filter by Stack

Find skills for your technology stack:

<CodeGroup>
```python React/TypeScript
list_skills(stack="react,typescript")
```

```python Python Backend
list_skills(stack="python,fastapi,postgresql")
```

```python Full Stack
list_skills(stack="nextjs,typescript,django,postgresql")
```

```python DevOps
list_skills(stack="docker,kubernetes,terraform")
```
</CodeGroup>

**How filtering works:**
1. Checks `compatibility.json` for explicit stack matches
2. Scans skill names for stack keywords
3. Ranks by relevance score (2 points for compatibility, 1 point for name match)
4. Returns sorted list

### Search by Keyword

Filter skills by name:

```python
# Find all React-related skills
list_skills(filter="react")

# Find caching skills
list_skills(filter="cache")

# Find testing skills
list_skills(filter="test")
```

## Understanding Compatibility

Each skill declares its compatibility in `compatibility.json`:

```json compatibility.json
{
  "languages": ["python"],
  "frameworks": ["fastapi"],
  "versions": {
    "fastapi": ">=0.100.0",
    "python": ">=3.10"
  },
  "dependencies": [
    "pydantic>=2.0",
    "uvicorn>=0.20"
  ],
  "platforms": ["linux", "macos", "windows"],
  "tags": ["api", "web", "async", "backend"]
}
```

### Checking Compatibility

Before applying a skill, verify it matches your environment:

```python
# Get skill metadata
get_skill("django_v5_best_practices")

# Look for version requirements in the skill header
# Example output:
# "Requires: Django >=5.0, Python >=3.11"
```

## Applying Skills to Your Project

<Steps>
  <Step title="Identify your need">
    What problem are you solving?
    
    ```python
    # Example: Need to add caching to API
    semantic_search("API response caching strategies")
    ```
  </Step>
  
  <Step title="Find relevant skills">
    Filter by your stack:
    
    ```python
    list_skills(stack="fastapi,redis")
    # Returns: redis_caching_patterns, fastapi_patterns
    ```
  </Step>
  
  <Step title="Fetch the skill">
    ```python
    get_skill("redis_caching_patterns")
    ```
  </Step>
  
  <Step title="Review compatibility">
    Check version requirements in the skill header.
  </Step>
  
  <Step title="Implement the pattern">
    Copy code examples and adapt to your project.
  </Step>
  
  <Step title="Test thoroughly">
    Skills include test patterns — use them!
  </Step>
</Steps>

## Skill Tiers

Skills are organized by quality tier:

- **L2 (Skills)**: Curated, validated, version-tagged
- **L3 (Truth)**: Empirically verified with evidence (50 items)
- **L4 (EUREKA)**: Breakthrough discoveries with measured ROI (383 items)
- **L5 (SOTA)**: State-of-the-art, currently unimprovable (140 items)

Access higher tiers:

```python
# Get validated truth patch
get_truth("qlora_myths")  # L3

# Get breakthrough discovery
get_eureka("semantic_cache_response")  # L4

# Get SOTA pattern (Pro tier)
get_aotc("mcp_streaming")  # L5
```

## Creating Your Own Skills

Skills are community-contributed. To add a skill:

<Steps>
  <Step title="Test in production">
    Only submit patterns you've validated in real projects.
  </Step>
  
  <Step title="Document thoroughly">
    Include code examples, edge cases, and version requirements.
  </Step>
  
  <Step title="Submit to MidOS">
    Open an issue or PR at [github.com/MidOSresearch/midos](https://github.com/MidOSresearch/midos).
  </Step>
  
  <Step title="Review process">
    MidOS maintainers validate, cross-check, and assign tier.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Skill not found">
    **Cause**: Typo in skill name or skill doesn't exist.
    
    **Fix**:
    ```python
    # List all skills first
    list_skills()
    
    # Then get exact name
    get_skill("exact_skill_name")
    ```
  </Accordion>
  
  <Accordion title="No skills for my stack">
    **Cause**: Stack not yet covered or using non-standard names.
    
    **Fix**:
    ```python
    # Try semantic search for patterns
    semantic_search("your framework patterns")
    
    # Request skill via GitHub issue
    # https://github.com/MidOSresearch/midos/issues/new
    ```
  </Accordion>
  
  <Accordion title="Skill code doesn't work">
    **Cause**: Version mismatch or missing dependencies.
    
    **Fix**:
    1. Check skill header for version requirements
    2. Verify your versions: `pip list` or `npm list`
    3. Install missing dependencies
    4. Report issue if skill is outdated
  </Accordion>
  
  <Accordion title="Stack filtering returns no results">
    **Cause**: Using abbreviations or non-canonical names.
    
    **Fix**:
    ```python
    # Wrong
    list_skills(stack="js,ts,pg")
    
    # Correct
    list_skills(stack="javascript,typescript,postgresql")
    ```
  </Accordion>
</AccordionGroup>
