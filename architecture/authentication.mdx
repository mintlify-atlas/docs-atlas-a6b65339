MidOS implements a **two-tier authentication system** with API key validation and rate limiting. This page covers the implementation details.

## Overview

The authentication system is implemented as FastMCP middleware in `modules/mcp_server/auth.py`.

**Key features**:
- **Two-tier access**: Community (free) vs Dev/Pro (API key required)
- **Rate limiting**: Per-tier query limits (100/mo → 25K/mo → 100K/mo)
- **API key format**: `midos_sk_` prefix (57 chars total)
- **Local development bypass**: Localhost gets dev tier without key
- **Security hardening**: Cloudflare proxy detection, path traversal protection

Reference: `modules/mcp_server/auth.py:1-526`

## Tier System (ACCESS_TIER_DOCTRINE v2.0)

MidOS uses a hierarchical tier system:

```
community (0) → dev (1) → ops (2) → admin (99)
```

| Tier | Level | Rate Limit | Tools Available |
|------|-------|------------|------------------|
| **community** | 0 | 100 queries/month | 8 basic tools |
| **dev** | 1 | 25,000 queries/month | All community + 8 advanced tools |
| **ops** | 2 | 100,000 queries/month | All dev tools |
| **admin** | 99 | Unlimited | All tools + admin ops |

**Example**: `midos_sk_a1b2c3d4e5f6789012345678901234567890abcdef123456`

**Length**: 57 characters total (9 prefix + 48 hex)

**Generation**:

### In-Memory Counter with Periodic Flush

To avoid excessive disk I/O, the middleware maintains **in-memory usage counters** and flushes to disk every 30 seconds:

```python
class ApiKeyMiddleware:
    def __init__(self):
        self._usage_mem: dict[str, int] = {}
        self._usage_month: str = _current_month()
        self._usage_flush_time: float = time.time()
    
    def _check_and_increment(self, identifier: str, tier: str):
        # Check in-memory counter
        count = self._usage_mem.get(identifier, 0)
        if count >= limit:
            return False, count, limit
        
        # Increment in memory
        self._usage_mem[identifier] = count + 1
        
        # Flush to disk every 30 seconds
        if (time.time() - self._usage_flush_time) > 30:
            self._flush_usage()
        
        return True, count + 1, limit
```

### Anonymous User Tracking

Unauthenticated users are tracked by **hashed IP address**:

def _get_anonymous_id(self) -> str:
    import hashlib
    try:
        headers = get_http_headers(include_all=True)
        ip = headers.get("cf-connecting-ip",
                         headers.get("x-real-ip", "anonymous"))
    except Exception:
        ip = "stdio"
    return f"anon_{hashlib.sha256(ip.encode()).hexdigest()[:16]}"

## Middleware Implementation

The `ApiKeyMiddleware` class hooks into FastMCP's middleware system:

```python
from fastmcp.server.middleware import Middleware, MiddlewareContext

class ApiKeyMiddleware(Middleware):
    async def on_call_tool(self, context: MiddlewareContext, call_next):
        """Gate tool execution by tier + rate limit."""
        tool_name = context.message.name
        tier, key = self._resolve_tier()
        
        # Check tier access
        if user_level < required_level:
            raise ToolError(f"'{tool_name}' requires {required_tier} tier")
        
        # Check rate limit
        allowed, count, limit = self._check_and_increment(identifier, tier)
        if not allowed:
            raise ToolError(f"Rate limit exceeded: {count}/{limit}")
        
        return await call_next(context)
```

## Authorization Header

Clients pass API keys via the `Authorization` header:

**Format**: `Bearer midos_sk_{key}`

**Example MCP config**:
```json
{
  "mcpServers": {
    "midos": {
      "url": "https://mcp.midos.dev/mcp",
      "headers": {
        "Authorization": "Bearer midos_sk_a1b2c3d4..."
      }
    }
  }
}
```

### Parsing Logic

```python
def _resolve_tier(self) -> tuple[str, str | None]:
    headers = get_http_headers(include_all=True)
    auth_header = headers.get("authorization", "")
    
    if not auth_header:
        return "community", None
    
    # Expect "Bearer midos_sk_..."
    parts = auth_header.split(" ", 1)
    if len(parts) != 2 or parts[0].lower() != "bearer":
        return "invalid", None
    
    token = parts[1].strip()
    if not token.startswith("midos_sk_"):
        return "invalid", token
    
    # Validate against stored keys
    keys = self._get_keys()
    key_info = keys.get(token)
    
    if not key_info or not key_info.get("active", False):
        return "invalid", token
    
    return key_info.get("tier", "pro"), token
```

## Security Features

### 1. Localhost Bypass (Development)

Requests from localhost get **dev tier without API key** for local development:

```python
def _is_localhost(self) -> bool:
    """Check if request originates from localhost.
    
    Only trusts X-Forwarded-For when request comes through Cloudflare.
    """
    headers = get_http_headers(include_all=True)
    local_addrs = {"127.0.0.1", "::1", "localhost"}
    
    # Trust X-Forwarded-For ONLY if CF-Connecting-IP is present
    cf_ip = headers.get("cf-connecting-ip", "")
    if cf_ip:
        forwarded = headers.get("x-forwarded-for", "").split(",")[0].strip()
        if forwarded in local_addrs:
            return True
    
    # Direct connection: check host header
    host = headers.get("host", "").split(":")[0]
    return host in local_addrs
```

Reference: `modules/mcp_server/auth.py:290-322`

### 2. Cloudflare Proxy Detection

X-Forwarded-For header is **only trusted when request comes through Cloudflare** (verified via `CF-Connecting-IP` header). This prevents IP spoofing.
